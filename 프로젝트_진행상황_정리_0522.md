## 2. 시도한 내용

### 방향 1 (초기): 텍스트 레이어 기반 API 번역
- pdf.js `getTextContent()`로 추출한 텍스트에서 용어집 미스 단어 클릭 시 Claude API 호출
- **포기 이유**: 교과서 PDF가 이미지 스캔본이라 텍스트 레이어 추출 불가

### 방향 2 (변경): Claude Vision 이미지 분석
1. PDF 페이지를 canvas에 렌더링 (기존 pdf.js 유지)
2. canvas → JPEG base64 변환 (900px 이하로 압축)
3. `api/translate.js` 서버리스 함수에 이미지 전송
4. Claude Vision API로 용어 위치(x, y, width, height, 0~1 비율) 추출
5. 반환된 좌표에 `.pdf-vocab-overlay` div 배치 → 탭 시 기존 툴팁 재사용
6. 용어집 미스 단어 클릭 시 `translate` action으로 번역 반환

### 구현된 코드 (롤백으로 현재 삭제 상태)

**`api/translate.js`** (신규)
- `action: 'analyze_image'` — 이미지 + 용어 목록 → Claude Vision → 좌표 JSON 반환
- `action: 'translate'` — 단어 + 언어 → 번역 + 정의 반환
- 모델: `claude-sonnet-4-20250514`

**`index.html`** 변경사항
- `renderPdfTextLayer()` → `analyzePageWithClaude()` + `renderVocabOverlays()`로 교체
- `pageAnalysisCache` — 동일 페이지 재분석 방지
- `showApiTooltip()` — 로딩 중 "번역 중..." → 결과/fallback 표시
- 데모 텍스트 비용어 토큰도 클릭 시 API 번역 호출

## 3. 실패 원인 분석

### 증상
- `/api/translate` 500 에러
- Vercel 로그: **Function Duration 7ms, No outgoing requests**
- → Claude API fetch 도달 전에 함수 종료

### 원인 추정 (확정 못함)
| 가능성 | 근거 |
|--------|------|
| `ANTHROPIC_API_KEY` 환경변수 미적용 | 7ms 종료 = API 키 체크에서 즉시 500 반환 패턴과 일치 |
| `req.body` 파싱 실패 | 수정 후에도 동일 증상 지속 |

**실제로 확인하지 못한 것**: 에러 응답 본문 (브라우저 Network 탭 Response 내용)

### 추가 시도한 디버깅
- `req.body` 안전 파싱 코드 추가
- 진단용 `console.log` 추가
- 진단용 `/api/debug` 엔드포인트 추가 → **이것이 사이트 전체 404 유발** (원인 불명)
- → 전체 롤백 결정

### 디버깅 방법 (다음번에는 이 순서로)
1. 브라우저 Network 탭 → `/api/translate` → **Response 탭** 에서 JSON 에러 메시지 직접 확인
2. 에러 메시지가 `"API 키가 설정되지 않았습니다."` → 환경변수 문제
3. 에러 메시지가 `"Claude API 오류 4xx: ..."` → 모델 ID 또는 키 값 문제

### 모델 ID 관련
