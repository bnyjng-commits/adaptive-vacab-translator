## 2. 완료된 내용

### PDF 텍스트 레이어 기반 렌더링 (v0.3에서 이어받음)
- `pdf.js getTextContent()`로 텍스트 추출 정상 작동 확인 (HWP→PDF 변환본 사용)
- 텍스트 레이어 스팬을 캔버스 위 투명 오버레이로 배치
- 용어집(`math_vocab_geometry.json`) 40개 수학 용어 + 20개 학습도구어 점선 밑줄 표시
- 용어 클릭 → 툴팁 번역 (EN/DE) 정상 작동

### Claude API 연결 (`api/translate.js` 신규)
- 용어집 미등록 한국어 단어 클릭 → `/api/translate` POST → Claude 번역 반환
- 모델: `claude-sonnet-4-6`, max_tokens: 200
- 툴팁에 "번역 중..." 로딩 상태 표시 후 결과 업데이트
- `ANTHROPIC_API_KEY` Vercel 환경변수 사용 (Production 설정 필요)

### PDF 고해상도 렌더링
- `window.devicePixelRatio` 반영
- `displayViewport`(CSS 표시 크기)와 `renderViewport`(고해상도 버퍼) 분리
- 레티나 디스플레이에서 선명하게 표시, 텍스트 레이어 클릭 정확도 유지

## 3. 미완료 — 다음 세션에서 해결할 것

### 단계별 번역이 시각적으로 적용되지 않음
**증상**: 단계 변경 시 한국어 원문이 그대로 보이면서 초록색 번역 배지(박스)만 추가로 떠있음

**의도한 동작**:
- 단계 1: 용어의 한국어 텍스트 자리에 번역어가 보여야 함 (수학 용어 + 학습도구어 모두)
- 단계 2: 학습도구어만 번역어로 대체, 수학 용어는 한국어 유지
- 단계 3: 수학 용어만 번역어로 대체, 학습도구어는 한국어 유지
- 단계 4: 모든 텍스트 한국어 원문 그대로

**현재 코드**: `has-vocab-translated` 클래스에 `color: var(--accent2)` + `background: var(--surface)` 로 원문을 덮는 방식 시도했으나 캔버스 위 투명 레이어 구조 때문에 시각적으로 반영 안 됨

**다음 시도 방향**: PDF 캔버스는 항상 한국어 원문을 보여주므로, 번역 시 해당 스팬에 불투명 흰 배경 + 번역어 텍스트가 실제로 덮어씌워지는지 브라우저에서 직접 확인 필요. `color: transparent` 상속 문제일 가능성 있음.

### 언어 전환 시 텍스트 대체도 동일하게 적용
- `setLang()` 호출 시 `rerenderPdfTextOnly()` 이미 연결되어 있으나 위 문제와 함께 미검증
