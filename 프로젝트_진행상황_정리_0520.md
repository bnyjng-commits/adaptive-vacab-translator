## 4. Supabase 테이블 구조

### students 테이블

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | uuid | 기본키, Supabase Auth user.id와 연결 |
| `name` | text | 학생 이름 (default: '') |
| `role` | text | 역할 |
| `current_stage` | int4 | 현재 번역 단계 (1~4) |
| `session_count` | int4 | 누적 수업 차시 |
| `stage_session_count` | int4 | 현재 단계에서의 차시 수 |
| `last_attended_date` | text | 마지막 수업 날짜 (YYYY-MM-DD). 로그인 시 오늘과 비교해 차시 자동 카운트 |
| `updated_at` | timestamp | 마지막 업데이트 시각 |

> ⚠️ 주의: 코드에서 `user_id` → `id`, `stage` → `current_stage` 로 사용해야 함

```sql
-- students 테이블에 컬럼 추가 (미적용 시 실행)
ALTER TABLE public.students
ADD COLUMN IF NOT EXISTS last_attended_date text;
```

### session_notes 테이블 (신규)

수업 종료 시 학생이 입력한 노트를 저장하는 테이블.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | bigint (auto) | 기본키 |
| `student_id` | uuid | students.id 참조 |
| `date` | text | 수업 날짜 (YYYY-MM-DD) |
| `note_text` | text | 학생이 입력한 노트 |
| `stage` | int | 노트 작성 시점의 번역 단계 |
| `created_at` | timestamp | 생성 시각 |

```sql
-- session_notes 테이블 생성 (미적용 시 실행)
CREATE TABLE public.session_notes (
  id bigint generated by default as identity primary key,
  student_id uuid references auth.users(id),
  date text not null,
  note_text text,
  stage int,
  created_at timestamp default now()
);

ALTER TABLE public.session_notes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "본인 삽입" ON public.session_notes
  FOR INSERT WITH CHECK (auth.uid() = student_id);

CREATE POLICY "본인 조회" ON public.session_notes
  FOR SELECT USING (auth.uid() = student_id);
```

## 5. Supabase RLS 정책 (적용 완료)

```sql
create policy "본인 조회" on public.students
  for select using (auth.uid() = id);

create policy "본인 삽입" on public.students
  for insert with check (auth.uid() = id);

create policy "본인 수정" on public.students
  for update using (auth.uid() = id);
```
