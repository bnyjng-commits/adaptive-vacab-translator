## 2. Supabase students 테이블 확정 구조

> ⚠️ 이전에 컬럼명을 여러 번 헷갈렸으니 이 표를 기준으로 사용할 것

| 컬럼명 | 타입 | 기본값 | 설명 |
|--------|------|--------|------|
| `id` | uuid | — | 기본키, Supabase Auth user.id와 동일 |
| `name` | text | — | 학생 이름 |
| `role` | text | — | 역할 (`'student'` 또는 `'teacher'`) |
| `current_stage` | int4 | 1 | 현재 번역 단계 (1~4) |
| `session_count` | int4 | 0 | 누적 수업 차시 |
| `stage_session_count` | int4 | 0 | 현재 단계에서의 연속 차시 수 |
| `updated_at` | timestamp | now() | 마지막 업데이트 |

**index.html에서 사용하는 Supabase 쿼리 패턴 (최종 확정)**

```javascript
// 조회
.from('students')
.select('name, current_stage, session_count, stage_session_count')
.eq('id', userId)

// 저장 (upsert — 없으면 insert, 있으면 update)
.from('students')
.upsert({ id: currentUser.id, current_stage: ..., ..., updated_at: new Date().toISOString() })
```

## 4. 오늘(0520) 한 일

### 구현한 기능 (v0.2)

**수업 시작 바** (stage bar 아래에 새로 추가)
- 로그인 후에만 표시 (비로그인 시 hidden)
- "이번 단계 X/3차시" + "누적 N차시" 실시간 카운터
- "수업 시작" 버튼 (태블릿 터치 기준 48px, 굵은 글씨)

**자동 페이딩 로직**
- 수업 시작 클릭 시: `session_count` +1, `stage_session_count` +1
- `stage_session_count >= 3`이고 4단계 미만이면 → 다음 단계 자동 전환, 카운터 1로 초기화
- 전환 시 토스트: `"단계가 올라갔어요! 이제 N단계예요 🎉"`
- 일반 시작 시 토스트: `"오늘 수업을 시작했어요 👋"` (3초 후 자동 사라짐)

**수동 단계 변경 시 초기화**
- 학생이 단계 버튼 직접 클릭 → `stage_session_count = 0` 초기화 + DB 저장

**에러 표시 개선**
- 저장 실패 시 빨간 토스트로 Supabase 오류 메시지 표시 (5초 유지)
- `supabaseClient` 또는 `currentUser` 없을 때도 안내 메시지 표시

**태블릿 UI 최적화**
- 모든 버튼 최소 높이 44~48px
- 데모 텍스트 줄간격 2.6
- 900px 중간 브레이크포인트 추가 (사이드패널 260px)
- 720px 이하: 수업 정보 숨김, 수업 시작 버튼 전체 너비

### 컬럼명 혼선 해결 과정

오늘 작업 중 Supabase 컬럼명이 예상과 달라 여러 번 수정했음. 최종 확정:

| JS 변수명 | Supabase 컬럼명 | 혼동 주의 |
|-----------|----------------|-----------|
| `currentStage` | `current_stage` | JS 변수와 다름 |
| `currentUser.id` | `id` | `user_id` 아님 |
| `stageSessionCount` | `stage_session_count` | 동일 |
| `sessionCount` | `session_count` | 동일 |

### update → upsert 변경 이유
- `update`는 행이 이미 있어야만 동작 → 첫 로그인 학생에게 저장 안 됨
- `upsert`는 없으면 INSERT, 있으면 UPDATE → 첫 로그인부터 정상 동작

## 5. v0.2 배포 테스트 전 체크리스트

> 다음 세션 시작 전에 이 항목부터 확인할 것

**Supabase RLS 정책 확인** (미확인 상태)

Supabase 대시보드 → Authentication → Policies → students 테이블에 아래 3개 있는지 확인:

```sql
-- 없으면 SQL Editor에서 실행
create policy "본인 조회" on public.students
  for select using (auth.uid() = id);

create policy "본인 삽입" on public.students
  for insert with check (auth.uid() = id);

create policy "본인 수정" on public.students
  for update using (auth.uid() = id);
```

**배포 후 테스트 순서**
1. `git push origin master:main` → Vercel 자동 배포

3. "수업 시작" 버튼 클릭
4. 에러 없으면: Supabase Table Editor에서 students 행 생성 확인
5. 빨간 에러 토스트 뜨면: 메시지 내용 기록 후 다음 세션에서 해결
